babylon-anyup

Seamless Z-up ‚Üî Y-up coordinate system conversion for Babylon.js

Stop wrestling with axis transformations. babylon-anyup is a lightweight plugin that automatically handles coordinate system conversion between Blizzard games (Warcraft 3, StarCraft 2), Blender, AutoCAD, and other Z-up workflows.

TypeScript

const plugin = new AnyUpPlugin({
  sourceSystem: 'z-up',
  targetSystem: 'y-up',
  autoConvert: true

The Challenge

Converting between these coordinate systems manually requires:

  • Swapping Y and Z coordinates for every vertex
  • Rotating models by -90¬∞ around X-axis
  • Recalculating normals and tangents
  • Adjusting camera and light directions
  • Hundreds of lines of error-prone transformation code

// Manual coordinate conversion (from EdgeCraft)
function convertZUpToYUp(vertex: Vector3): Vector3 {
  return new Vector3(
    vertex.x,
    vertex.z,  // Swap Y and Z
    -vertex.y
  );
}

// Apply to every vertex
mesh.getVerticesData(VertexBuffer.PositionKind).forEach((v, i) => {
  if (i % 3 === 0) {
    const x = vertices[i];
    const y = vertices[i + 1];
    const z = vertices[i + 2];
    vertices[i] = x;
    vertices[i + 1] = z;
    vertices[i + 2] = -y;
  }
});

// Rotate model
mesh.rotation.x = -Math.PI / 2;

How It Works

üîÑ Transformation Strategy

Uses a strategy pattern to apply coordinate system transformations. Each system (Y-up, Z-up) has its own optimized transformation strategy.

üéØ Scene Integration

Hooks into Babylon.js scene lifecycle to automatically transform meshes as they're added. No manual intervention required.

‚ö° Performance

Transformations are applied once during scene initialization. Zero runtime overhead after initial load.

üíæ Optional Preservation

Can optionally preserve original transforms in mesh metadata for debugging or analysis.
